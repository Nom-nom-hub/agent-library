## Research Process

1. **Define the Question**
   - Clarify what needs to be evaluated
   - Identify decision criteria
   - Note constraints and requirements
   - Set research scope

2. **Gather Information**
   - Search documentation and guides
   - Review community discussions
   - Check benchmarks and performance data
   - Identify available solutions

3. **Evaluate Options**
   - Compare feature sets
   - Check performance characteristics
   - Review community size and support
   - Assess learning curve
   - Check licensing and compatibility

4. **Document Findings**
   - Create comparison table
   - Document pros/cons for each option
   - Explain trade-offs
   - Provide recommendations

5. **Make Recommendation**
   - Summarize findings
   - Recommend best option for context
   - Explain reasoning
   - List caveats and considerations

## Research Documentation

Include:
- Problem statement
- Evaluation criteria
- Options evaluated
- Feature comparison table
- Performance data
- Community/support assessment
- Recommendation with justification
- Alternative options and trade-offs
- Links to sources

## Evaluation Criteria

Common factors to consider:
- Functionality and features
- Performance and scalability
- Ease of integration
- Learning curve and documentation
- Community size and activity
- Support and maintenance
- License and compatibility
- Cost (if applicable)
- Industry adoption

## Success Criteria

- [ ] Research question clearly answered
- [ ] Options systematically compared
- [ ] Data sources cited
- [ ] Trade-offs documented
- [ ] Recommendation justified
- [ ] Alternative approaches noted
